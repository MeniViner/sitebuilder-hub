| סביבת יעד / `environment` | Basic | Operational environment for caution, reporting and filtering. |

| מספר אישי / `ownerPersonalNumber` | Basic | Required for create-new owner/admin seed identity. |

| טלפון / `ownerPhone` | Basic | Optional contact detail only. |

| פריסה ראשונית | Plan/deploy step | Blocked until Create New Site has created/verified `siteDB`, `siteUsersDb`, `siteAssets`, `dist`, and runtime config. Browser Deploy is only the final initial deploy step after provisioning. |

## UI Changes

- Basic mode now shows only owner decisions: site name, code/path, Site Builder siteId, environment/storage type, SharePoint URL, Builder backend URL, owners and initial admins.
- Advanced settings now collapse technical fields: site libraries, bootstrap, runtime config path, credential reference, safe collection, Mongo metadata, widgets mapping and connector mode.
- Generated defaults are shown as readable preview instead of asking the owner to invent infrastructure values.
- Mongo plan details use Hebrew labels and humanized blockers. Raw JSON is only inside `פרטים טכניים`.
- The plan preview is grouped into `SharePoint hosting`, `Mongo backend`, `Deploy`, and `לא אוטומטי`.

## Validation Changes

- Required-field messages now explain what is missing, why it matters and how to fix it.
- Builder backend URL validates URL shape and explains that it is not SharePoint.
- Credential reference warns against raw API key input.
- `safeCollectionName` can remain empty for auto-generation; explicit values get a safe-name format check.
- Absolute runtime config paths must stay under the SharePoint site root.
- Duplicate physical/runtime identity is explained as different from duplicate `siteCode`.
